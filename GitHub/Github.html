Git & GitHub

Git

Hva er forskjellen mellom Git og GitHub?
Git er en type programvare som sjekker og holder kontroll på versjoner av filer man har på PCen.
Den gir også veldig bra tilbakemelding på filer til flere som jobber med samme fil.
Git er mest brukt for "Source Code Management" i programutvikling. Fordelene med både Git og GitHub er at de er
utrolig raske, lett å bruke, og at du kan oppdatere lokalt. Dette er et system utviklere bruker til å legge ut nye versjoner
av sin Beta eller sitt program.


GitHub

GitHub er en Web repository system der man legger ut sine filer og prosjekter.
Her deler alle utviklere sine prosjekter så alle kan lære av hverandre.
Når endringer er gjort i GitHub står det i loggen så man husker hva som ble endret.
Den inneholder alle funksjonene som Git har, men også noen unike funksjoner. Git er en streng kommando-linje verktøy
mens GitHub gir brukeren mulighet til å bruke repository både på desktop og mobil.
GitHub gir brukeren også et valg om å ha en gratis brukerkonto og "VIP" der man kan legge ut private repositories.
Flere funksjoner som GitHub har er "Bug tracking" og "task management" for alle prosjekter man legger ut, GitHub er også en av de støreste "code host" i verden.
Både Git og Github er "Open Source" systemer for kode komiteen.
